# Milestones

このプロジェクトを段階的に進めるためのマイルストーン一覧です。  
MVPで「最低限使える状態」に到達したあと、絵本記録機能や自動化を追加していく想定です。

---

## Milestone 0: リポジトリ・設計ドキュメントの整備

**目的:** 開発の土台づくり

- [ ] GitHub リポジトリ `yomikikase-planner` を作成
- [ ] README / Milestone / ArchitectureDesign をコミット
- [ ] 使用する技術スタックの暫定決定
- [ ] ベースのディレクトリ構成・Lint/Format 設定（ESLint, Prettier 等）

---

## Milestone 1: ドメインモデル & DB スキーマ設計

**目的:** 年度単位の活動と、人・アンケート・絵本の関係を整理する

- [ ] 年度 (`SchoolYear`) の概念をモデル化
- [ ] ユーザー (`User` / `UserYearProfile`) モデルを定義
- [ ] アンケート (`Survey`, `SurveyDate`, `Response`, `ResponseDetail`) モデルを定義
- [ ] 確定日程 (`ConfirmedEvent`, `ConfirmedParticipant`) モデルを定義
- [ ] 絵本 (`Book`, `BookReadingRecord`) モデルを定義
- [ ] マイグレーションツール導入（Prisma / Knex / Sequelize など）

---

## Milestone 2: 管理画面の MVP（アンケート作成・結果確認）

**目的:** 現在の Googleフォーム+Excel 運用を、まずは管理画面で置き換える

- [ ] 管理者ログイン（簡易認証でOK）
- [ ] アンケート一覧画面
- [ ] アンケート作成画面
  - [ ] タイトル・説明
  - [ ] 年度
  - [ ] 開催候補日（日時＋学年）
- [ ] アンケート結果一覧画面
  - [ ] 回答一覧表示（名前・学年・クラス・各日程の可否）
  - [ ] 学年・クラスでフィルタ
  - [ ] CSV エクスポート

---

## Milestone 3: 確定日程・参加者登録の実装

**目的:** 「決まった情報をどこかに記録しておく」をアプリで完結させる

- [ ] 確定日程一覧画面
  - [ ] 日付・年度・学年・クラス・参加者の表示
- [ ] 確定日程編集画面
  - [ ] 年度・日付・学年・クラスの入力
  - [ ] 参加者選択（学年・クラスでフィルタ）
- [ ] アンケート回答のユーザーを起点に候補者を出せるようにする（あると便利）

※ この時点では「自動割当ロジック」は入れず、別ツールで検討した結果を手入力する運用を前提とする。

---

## Milestone 4: LIFF ミニアプリによるアンケート回答

**目的:** Google フォームを卒業して、LINE 上でアンケートを完結させる

- [ ] LINE Developers でチャネル・LIFF アプリを作成
- [ ] LIFF から `userId` を取得し、バックエンドに連携
- [ ] 初回アクセス時のプロフィール登録画面
  - [ ] 名前
  - [ ] 学年
  - [ ] クラス
- [ ] アンケート回答画面
  - [ ] 対象アンケートの候補日一覧を表示
  - [ ] 各日程の参加可否を送信
- [ ] 管理画面のアンケート結果と、LIFFからの回答を連携

---

## Milestone 5: 絵本記録機能の実装（基本版）

**目的:** 「どの年度・学年で、どの絵本を読んだか」を長期的に活用できるようにする

- [ ] 絵本マスタ登録画面（管理者）
  - [ ] ISBN 入力 → 書誌情報取得（外部 API 連携）
  - [ ] タイトル等の編集
- [ ] 読み聞かせ記録登録
  - [ ] 日付・年度・学年・クラス・絵本を紐付けて保存
- [ ] 年度・学年での「読んだ絵本一覧」画面
- [ ] 本ごとの「どの年度・学年で読んだか」履歴画面

---

## Milestone 6: データライフサイクル & 引き継ぎ

**目的:** 「いつか卒園する」ことを前提に、きれいに畳める / 引き継げる設計にする

- [ ] 年度ごとのデータ削除機能
  - [ ] アンケート・回答・確定日程・UserYearProfile の削除
- [ ] 匿名統計 or 完全削除の方針を文書化
- [ ] 管理者アカウントの追加・削除機能（後任への引き継ぎ）
- [ ] 簡易運用マニュアルの作成（別ドキュメント）
