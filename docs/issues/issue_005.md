# Issue 005: LIFF MVP 画面フロー

## 目的

LIFF で `初回登録 -> アンケート表示 -> 回答送信` を一連で完了できる画面を作る。

## 担当

- Primary: Agent

## 依存関係

- [issue_002.md](./issue_002.md)
- [issue_004.md](./issue_004.md)

## 対象ファイル

- [liff/index.html](../../liff/index.html)

## 完了条件

- [ ] `/api/config` と URL パラメータから LIFF ID を解決できる
- [ ] `liff.init()` とログイン遷移が動作する
- [ ] LIFF の ID トークンを取得し、認証付き API に渡せる
- [ ] 初回アクセス時はプロフィール登録フォームを表示する
- [ ] 登録済みユーザーはフォームをスキップしてアンケート表示へ進む
- [ ] アクティブアンケートのタイトル、説明、候補日一覧を表示できる
- [ ] 各候補日に対して `可`, `不可`, `未定` のいずれかを選択できる
- [ ] 全候補日に未回答がある場合は送信を拒否する
- [ ] 送信成功時に完了メッセージを表示する
- [ ] API エラー時にユーザーが判別できるエラーメッセージを表示する

## 確認方法

- [ ] 未ログイン状態からログイン完了まで確認する
- [ ] 未登録ユーザーでプロフィール登録ができることを確認する
- [ ] 登録済みユーザーでアンケート画面へ直接遷移することを確認する
- [ ] 全候補日に回答して送信できることを確認する
- [ ] 一部未回答で送信した場合にエラー表示されることを確認する

## 非スコープ

- 既存回答の事前表示
- リッチな管理 UI
- デザインシステム化
